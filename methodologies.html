Agile Methodologies

The Agile Manifesto

The Agile Manifesto emphasizes individuals, interactions, working software, customer collaboration, and responsiveness to change.

Common Practices

Pair Programming (PP)

Pair programming involves two developers working together on the same computer. One writes code (driver), while the other reviews it (observer). They swap roles periodically. Benefits include reduced errors, better design, and learning opportunities. However, it can be costly and may not suit everyone.

Test-Driven Development (TDD)

TDD involves writing test cases before coding. The process includes:

  1. Add a new test.
  2. Write code to pass the test.
  3. Run all tests to ensure they pass.
  4. Refactor the code for optimization.

This approach ensures code meets requirements and avoids unnecessary development.

Scrum

Scrum is an iterative and incremental framework for managing complex projects.

  • Roles: ScrumMaster, Product Owner, and Developers.
  • Process: Features are described as user stories and organized in a backlog. Development occurs in iterations (sprints), typically 2-4 weeks long.
  • Meetings: Planning meetings, daily standups, and review meetings after each sprint.

Traditional Methodologies

Rational Unified Process (RUP)

RUP is a framework for software development that emphasizes iterative development, use cases, and risk management.

Microsoft Solutions Framework (MSF)

MSF is a flexible framework for managing projects and processes, often used for enterprise-scale software solutions.
